## Story

Como desenvolvedor do Nexus ROAS,
Quero configurar TypeScript e um pipeline de build com esbuild,
Para que todos os scripts de pixel sejam escritos em TypeScript e compilados individualmente para JS minificado, substituindo os arquivos `.txt` manuais.

---

## Acceptance Criteria

- [ ] AC1: `tsconfig.json` configurado no `worker/` com `strict: true`, `target: ES2020`, `module: ESNext`
- [ ] AC2: `esbuild.config.ts` com 5 entry points independentes (um por script de pixel)
- [ ] AC3: Cada entry point gera um arquivo `dist/{name}.js` minificado com placeholder de configuração via `banner`
- [ ] AC4: Scripts npm disponíveis: `build:pixel` (todos), `build:watch` (modo dev), `build:shopify-storefront`, `build:shopify-checkout`, `build:cartpanda-checkout`, `build:yampi-checkout`, `build:direct-traffic`
- [ ] AC5: Os 3 arquivos `.txt` existentes (`pixel-template.txt`, `yampi-checkout-template.txt`, `cartpanda-checkout-template.txt`) são removidos após build bem-sucedido
- [ ] AC6: Build completo roda em < 5 segundos
- [ ] AC7: `npm run build:pixel` gera os 5 arquivos em `worker/dist/` sem erros

## Dev Notes

### Estrutura de Placeholders por Script

O mecanismo de placeholder permite ao worker injetar configuração de projeto em runtime sem recompilar:

```typescript
// esbuild.config.ts — exemplo de banner por entry point
const banners = {
  'shopify-storefront': '/*__CONFIG__*/\n/*__NX_USER__*/',
  'shopify-checkout':   '/*__NX_COLLECT__*/\n/*__META_PIXEL_IDS__*/\n/*__TIKTOK_PIXEL__*/\n/*__GA4_ID__*/\n/*__META_TEST__*/\n/*__TIKTOK_TEST__*/',
  'cartpanda-checkout': '/*__NX_COLLECT__*/\n/*__META_PIXEL_IDS__*/\n/*__TIKTOK_PIXEL__*/\n/*__GA4_ID__*/\n/*__META_TEST__*/\n/*__TIKTOK_TEST__*/',
  'yampi-checkout':     '/*__NX_COLLECT__*/\n/*__META_PIXEL_IDS__*/\n/*__TIKTOK_PIXEL__*/\n/*__GA4_ID__*/\n/*__META_TEST__*/\n/*__TIKTOK_TEST__*/',
  'direct-traffic':     '/*__CONFIG__*/\n/*__NX_COLLECT__*/\n/*__META_PIXEL_IDS__*/\n/*__TIKTOK_PIXEL__*/\n/*__GA4_ID__*/\n/*__META_TEST__*/\n/*__TIKTOK_TEST__*/',
}
```

### tsconfig.json Referência

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "lib": ["ES2020", "DOM"],
    "strict": true,
    "noEmit": true,
    "moduleResolution": "bundler",
    "skipLibCheck": true
  },
  "include": ["src/**/*.ts"],
  "exclude": ["node_modules", "dist"]
}
```

### Contexto dos .txt removidos

Os arquivos `.txt` existentes são JS minificado com comentários-placeholder. O build TypeScript os substitui completamente. O worker (`pixel.js` atual) precisa ser atualizado na story 5.1 para servir os novos arquivos de `dist/` em vez dos `.txt`.
