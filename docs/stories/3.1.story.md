## Story

Como script de pixel rodando no browser,
Quero um módulo que gerencie a integração com Meta (Facebook) Pixel,
Para que `fbevents.js` seja carregado uma única vez, múltiplos pixels sejam inicializados com Advanced Matching, e eventos sejam disparados com `event_id` para deduplicação com o CAPI.

---

## Acceptance Criteria

- [ ] AC1: `worker/src/pixel/platforms/meta.ts` exporta interface `MetaModule`
- [ ] AC2: `init(pixelIds: string[], nxUser: string, geo?: GeoData)` carrega `fbevents.js` uma única vez e inicializa todos os pixels
- [ ] AC3: Advanced Matching inclui: `external_id` (nx_user), geo hash (city, state, country, zip) se disponível
- [ ] AC4: `autoConfig` desabilitado: `fbq('set', 'autoConfig', false, pixelId)`
- [ ] AC5: `fire(eventName: EventName, eventId: string, customData?: CustomData, signals?: SignalMap)` dispara para todos os pixels inicializados
- [ ] AC6: Mapeamento de eventos Nexus → Meta: `PageView`, `ViewContent`, `AddToCart`, `InitiateCheckout`, `AddPaymentInfo`, `Purchase`, `Lead`, etc.
- [ ] AC7: `testEventCode` aplicado quando presente na config
- [ ] AC8: Falha silenciosa se `window.fbq` não disponível
- [ ] AC9: Suporte a `meta_pixel_ids_mirror` — pixels espelho disparados junto com o principal

### Advanced Matching com geo

```typescript
function buildAdvancedMatching(nxUser: string, geo?: GeoData): Record<string, string> {
  const am: Record<string, string> = {}
  if (nxUser) am.external_id = nxUser
  if (geo?.city)    am.ct = geo.city.toLowerCase().replace(/[^a-z]/g, '')
  if (geo?.region)  am.st = geo.region.toLowerCase().replace(/[^a-z0-9]/g, '').substring(0, 2)
  if (geo?.postal)  am.zp = geo.postal.replace(/[\s-]/g, '')
  if (geo?.country) am.country = geo.country.toLowerCase().replace(/[^a-z]/g, '').substring(0, 2)
  return am
}
```
