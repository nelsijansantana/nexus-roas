## Story

Como worker que processa webhooks de múltiplos gateways,
Quero todos os 14 parsers de gateway em TypeScript com tipagem completa,
Para que cada parser tenha um contrato claro, HMAC validation tipada, e o retorno seja um `ParsedGatewayEvent` padronizado.

---

## Acceptance Criteria

- [ ] AC1: Interface `ParsedGatewayEvent` definida com todos os campos padronizados extraídos de qualquer gateway
- [ ] AC2: Interface `GatewayParser` com método `parse(body, headers, secret): ParsedGatewayEvent`
- [ ] AC3: Todos os 14 parsers implementam `GatewayParser`:
  - `shopify`, `cartpanda`, `yampi` (já na 6.1), `hotmart`, `kiwify`, `eduzz`, `hubla`, `greenn`, `kirvano`, `lastlink`, `pagtrust`, `payt`, `perfectpay`, `ticto`
- [ ] AC4: Cada parser extrai: `order_id`, `customer`, `items`, `value`, `currency`, tracking signals
- [ ] AC5: HMAC validation implementada nos gateways que suportam (`shopify`, `cartpanda`, `yampi`, `kiwify`)
- [ ] AC6: Factory `getParser(gateway: string): GatewayParser` retorna o parser correto
- [ ] AC7: Sem `any` explícito — tipos específicos por gateway

## Dev Notes

### Interface ParsedGatewayEvent

```typescript
interface ParsedGatewayEvent {
  gateway: string
  order_id: string
  pixel_id: string

  // Valor
  value: number
  currency: string

  // Cliente
  customer: ParsedCustomer

  // Itens
  items: ParsedItem[]

  // Sinais de tracking (extraídos do payload)
  nx_user?: string
  fbclid?: string
  fbc?: string
  fbp?: string
  ttclid?: string
  ttp?: string
  ga_client_id?: string
  cart_token?: string

  // UTMs
  utm_source?: string
  utm_medium?: string
  utm_campaign?: string
  utm_content?: string
  utm_term?: string

  // Metadados
  event_id: string          // determinístico: purchase_{pixelId}_{orderId}
  event_time: number        // unix timestamp
  raw?: unknown             // payload original (para debug)
}

interface ParsedCustomer {
  email?: string
  phone?: string            // E.164
  first_name?: string
  last_name?: string
  city?: string
  state?: string
  zip?: string
  country?: string
  ip?: string
}

interface ParsedItem {
  id: string
  name?: string
  price: number
  quantity: number
}
```

### Shopify parser — note_attributes

```typescript
// Shopify traz sinais em order.note_attributes (array de {name, value})
const attrs = Object.fromEntries(
  (order.note_attributes || []).map(({ name, value }) => [name, value])
)
const nx_user = attrs['_nx_user'] || attrs['nx_user']
```

### CartPanda parser — metadata similar ao Yampi

CartPanda também tem `metadata.data` com estrutura similar ao Yampi.

### Factory

```typescript
const PARSERS: Record<string, GatewayParser> = {
  shopify:     new ShopifyParser(),
  cartpanda:   new CartPandaParser(),
  yampi:       new YampiParser(),
  hotmart:     new HotmartParser(),
  // ...
}

export function getParser(gateway: string): GatewayParser {
  const parser = PARSERS[gateway.toLowerCase()]
  if (!parser) throw new Error(`Unknown gateway: ${gateway}`)
  return parser
}
```
