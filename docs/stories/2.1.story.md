## Story

Como desenvolvedor do Nexus ROAS,
Quero um arquivo central de tipos TypeScript para o sistema de tracking,
Para que todos os módulos compartilhem contratos de dados consistentes e o compilador detecte incompatibilidades em build-time.

---

## Acceptance Criteria

- [ ] AC1: `worker/src/pixel/core/types.ts` exporta todas as interfaces e tipos centrais
- [ ] AC2: Interface `NexusEvent` cobre todos os campos enviados ao endpoint `/event`
- [ ] AC3: Interface `SignalMap` cobre todos os click IDs, cookies de plataforma e UTMs rastreados
- [ ] AC4: Interface `PixelConfig` cobre todos os campos de configuração injetados via placeholder
- [ ] AC5: Interface `GeoData` para dados de geolocalização do usuário
- [ ] AC6: Tipos de eventos (`EventName`) como union type ou enum cobrindo todos os eventos suportados
- [ ] AC7: Nenhum `any` explícito — todos os tipos são específicos
- [ ] AC8: Arquivo compila sem erros com `tsc --noEmit`

### Campos NexusEvent (baseado nos payloads reais dos sistemas de referência)

```typescript
interface NexusEvent {
  event: EventName
  event_id: string
  nx_user: string
  page_url: string
  page_title?: string
  page_referrer?: string
  session_id?: string
  browser_data: BrowserData
  utm_data?: Partial<UtmData>
  user_data?: UserData
  custom_data?: CustomData
  test_event_code?: string
```
