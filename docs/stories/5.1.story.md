## Story

Como worker Cloudflare,
Quero um handler que sirva os scripts JS compilados com configuração injetada em runtime,
Para que cada projeto receba seu script personalizado com pixels, collect_url e test codes corretos, substituindo o mecanismo antigo baseado em `.txt`.

---

## Acceptance Criteria

- [ ] AC1: `GET /scripts/:name.js?pixel_id=XXXX` serve o arquivo correto de `dist/`
- [ ] AC2: Worker lê configuração do projeto (D1) usando `pixel_id`
- [ ] AC3: Substitui placeholders no JS com valores do projeto:
  - `/*__CONFIG__*/` → `const __CONFIG__ = {...};`
  - `/*__NX_USER__*/` → `const __NX_USER__ = "";` (vazio por padrão, preenchido se loja Shopify)
  - `/*__NX_COLLECT__*/` → URL da collect endpoint como string
  - `/*__META_PIXEL_IDS__*/` → array JSON de pixel IDs
  - `/*__TIKTOK_PIXEL__*/` → pixel ID como string ou `null`
  - `/*__GA4_ID__*/` → measurement ID como string ou `null`
  - `/*__META_TEST__*/` → test event code ou `""`
  - `/*__TIKTOK_TEST__*/` → test event code ou `""`
- [ ] AC4: `Content-Type: application/javascript` com `Cache-Control: public, max-age=300`
- [ ] AC5: Scripts válidos: `shopify-storefront`, `shopify-checkout`, `cartpanda-checkout`, `yampi-checkout`, `direct-traffic`
- [ ] AC6: Retorna `404` para nomes de script não reconhecidos
- [ ] AC7: Retorna `400` se `pixel_id` ausente ou inválido

---

## Scope

**IN:**
- Servir arquivos `dist/*.js` com injeção de config runtime
- Substituição de todos os placeholders
- Cache headers

**OUT:**
- Geração/compilação dos scripts (story 1.1)
- Lógica de autenticação do projeto
- Leitura de `nx_user` server-side para injeção (feature opcional, fora do escopo inicial)

## Dev Notes

### Injeção de config

```typescript
function injectConfig(js: string, project: Project): string {
  const config = {
    collect_url: `https://worker.example.com/event`,
    meta_pixel_id: project.meta_pixel_id,
    meta_pixel_ids_mirror: project.meta_pixel_ids_mirror || [],
    tiktok_pixel_id: project.tiktok_pixel_id,
    ga4_measurement_id: project.ga4_measurement_id,
    google_ads_conversion_id: project.google_ads_conversion_id,
    meta_test_event_code: project.meta_test_event_code,
    tiktok_test_event_code: project.tiktok_test_event_code,
    debug: false,
  }

  return js
    .replace('/*__CONFIG__*/', `const __CONFIG__ = ${JSON.stringify(config)};`)
    .replace('/*__NX_USER__*/', `const __NX_USER__ = "";`)
    .replace('/*__NX_COLLECT__*/', JSON.stringify(config.collect_url))
    .replace('/*__META_PIXEL_IDS__*/', JSON.stringify(
      [project.meta_pixel_id, ...(project.meta_pixel_ids_mirror || [])].filter(Boolean)
    ))
    .replace('/*__TIKTOK_PIXEL__*/', JSON.stringify(project.tiktok_pixel_id || null))
    .replace('/*__GA4_ID__*/', JSON.stringify(project.ga4_measurement_id || null))
    .replace('/*__META_TEST__*/', JSON.stringify(project.meta_test_event_code || ''))
    .replace('/*__TIKTOK_TEST__*/', JSON.stringify(project.tiktok_test_event_code || ''))
}
```

### Servindo arquivos dist/

Os arquivos `dist/*.js` podem ser servidos de duas formas:
1. **KV Storage** — upload no deploy (mais performático)
2. **Import inline** — importar como string no worker bundle (mais simples)

Para a implementação inicial, usar import inline via esbuild `--loader:.js=text`.
