## Story

Como script de pixel rodando no browser,
Quero um módulo de identidade que gerencie o `nx_user`,
Para que cada visitante tenha um identificador persistente e consistente entre sessões e domínios.

---

## Acceptance Criteria

- [ ] AC1: `worker/src/pixel/core/identity.ts` exporta função `getOrCreateNxUser(): string`
- [ ] AC2: Formato do `nx_user`: `nxu_` + 13 caracteres alfanuméricos (ex: `nxu_kdkl543813h7`)
- [ ] AC3: Prioridade de leitura: cookie `nx_user` → cookie `nx_lid` → `localStorage.nx_user` → `localStorage.nx_lid` → URL params `src` ou `sck` → geração de novo
- [ ] AC4: Persistência: cookie (1 ano, `SameSite=Lax; Secure`) + `localStorage`
- [ ] AC5: Resolução de conflito: quando dois `nx_user` coexistem, vence o mais recente (timestamp embutido no prefixo)
- [ ] AC6: Cookie `nx_lid` escrito com `domain` cross-subdomain (ex: `.myshopify.com`) para funcionar em subdomínios
- [ ] AC7: Graceful degradation: se `localStorage` não disponível (bloqueado), funciona apenas com cookies
- [ ] AC8: Função `isNxUser(value: string): boolean` para validação de formato

## Dev Notes

### Formato nx_user

O timestamp está embutido para permitir resolução de conflito:
```
nxu_kdkl543813h7
     ^^^^^^^^^^^
     random (não há timestamp explícito — usa order-of-creation implícita)
```

Na implementação de referência (`pixel-template.txt`):
```javascript
function getTimestamp(nxUser) {
  const ts = parseInt((nxUser || '').split('-')[0], 10)
  return isNaN(ts) ? 0 : ts
}
// Prefixo com timestamp:
const newer = getTimestamp(serverUser) > getTimestamp(localUser)
```

**Nota:** O formato atual NÃO tem timestamp explícito. Ao implementar, manter compatibilidade com formato existente `nxu_XXXXX`. Para resolução de conflito, o valor do servidor (injetado via `__NX_USER__`) vence se não houver local.

### Cookie nx_lid cross-subdomain

```typescript
function setNxLid(value: string): void {
  const parts = window.location.hostname.split('.')
  const baseDomain = parts.length >= 2 ? '.' + parts.slice(-2).join('.') : window.location.hostname
  document.cookie = `nx_lid=${encodeURIComponent(value)}; max-age=${365*24*60*60}; path=/; domain=${baseDomain}; SameSite=Lax; Secure`
}
```

### Referência nos sistemas de referência

Em `tracker-agent/apps/worker/pixel/index.ts`:
- `NexusPublisher` gerencia `nx_user`
- Sincroniza com Shopify via `/cart/update.js`

Em `pixel-template.txt` (atual):
- `nx_user` lido de localStorage com fallback para cookie
- Injeção server-side via `__NX_USER__` para lojas Shopify
