## Story

Como queue processor que despacha conversões,
Quero um dispatcher de Google Ads Offline Conversion,
Para que compras com `gclid`, `gbraid` ou `wbraid` sejam reportadas ao Google Ads quando configurado no projeto.

---

## Acceptance Criteria

- [ ] AC1: `worker/src/capi/gads.ts` exporta `dispatchGoogleAds(event, project): Promise<GAdsResult>`
- [ ] AC2: Dispara apenas quando `gclid` OU `gbraid` OU `wbraid` presente no evento
- [ ] AC3: Retorna `{ success: true, skipped: true }` (sem erro) quando nenhum Google click ID disponível
- [ ] AC4: Integração via `gtag` conversion (Google Ads Conversion Tracking API)
- [ ] AC5: `conversion_action` mapeada por `google_ads_events.purchase` do projeto
- [ ] AC6: `value` e `currency` incluídos quando disponíveis
- [ ] AC7: `transaction_id` = `order_id` para deduplicação
- [ ] AC8: Retorna `{ success, error? }` sem lançar exceção

---

## Scope

**IN:**
- Dispatch de conversão para Google Ads quando gclid/gbraid/wbraid presente
- Skip graceful quando sem click IDs Google

**OUT:**
- Google Ads Enhanced Conversions (escopo futuro)
- Google Ads API direta (usar gtag por ora)
