## Story

Como script de pixel rodando no browser,
Quero um módulo que gerencie a integração com TikTok Pixel,
Para que `ttq.js` seja carregado, o pixel inicializado com `identify()`, e eventos disparados com `event_id` para deduplicação com o TikTok Events API.

---

## Acceptance Criteria

- [ ] AC1: `worker/src/pixel/platforms/tiktok.ts` exporta interface `TikTokModule`
- [ ] AC2: `init(pixelId: string, nxUser: string)` carrega `ttq.js` e chama `ttq.identify({ external_id: nxUser })`
- [ ] AC3: `fire(eventName: EventName, eventId: string, customData?: CustomData, signals?: SignalMap)` dispara evento mapeado
- [ ] AC4: `PageView` dispara via `ttq.page({}, { event_id })` (não `ttq.track`)
- [ ] AC5: Mapeamento de eventos Nexus → TikTok (ver tabela de referência)
- [ ] AC6: `ttq.load(pixelId, { test_event_code })` quando test code presente
- [ ] AC7: `ttclid` e `ttp` incluídos no customData quando disponíveis
- [ ] AC8: Falha silenciosa se `window.ttq` não disponível
- [ ] AC9: Guard contra carregamento duplo do script `ttq.js`

```typescript
// PageView é especial no TikTok — usa ttq.page() não ttq.track()

  window.ttq.page({}, { event_id: eventId })
} else {
  window.ttq.track(tiktokEvent, customData, { event_id: eventId })
}
```
