## Dev Notes

### Diferença crítica vs CartPanda: URL params são obrigatórios

A Yampi é sempre acessada via domínio diferente da origem (Shopify ou LP). Não há cookies cross-domain. Os sinais chegam **exclusivamente via URL params** decorados pelo `shopify-storefront.ts` ou `direct-traffic.ts`.

```typescript
function readAndPersistSignals(): Record<string, string> {
  const params = new URLSearchParams(window.location.search)
  const signals: Record<string, string> = {}

  const SIGNAL_KEYS = [
    'src', 'sck', 'fbclid', 'gclid', 'gbraid', 'wbraid', 'ttclid', 'msclkid',
    'utm_source', 'utm_medium', 'utm_campaign', 'utm_content', 'utm_term', 'utm_id',
  ]

  SIGNAL_KEYS.forEach(key => {
    const val = params.get(key)
    if (val) {
      signals[key] = val
      // Persistir em cookie para eventos subsequentes (sem recarregar a página)
      try {
        document.cookie = `_nx_${key}=${encodeURIComponent(val)}; max-age=${30*24*60*60}; path=/; SameSite=Lax`
      } catch {}
    } else {
      // Fallback para cookie persistido anteriormente
      const fromCookie = readCookie(`_nx_${key}`)
      if (fromCookie) signals[key] = fromCookie
    }
  })

  return signals
}
```

### Resolução de nx_user (Yampi)

```typescript
function resolveNxUser(signals: Record<string, string>): string {
  // 1. URL params (prioridade absoluta — cross-domain)
  const fromUrl = signals['src'] || signals['sck']
  if (fromUrl && fromUrl.startsWith('nxu_')) return fromUrl

  // 2. Cookie local (se já visitou antes via mesmo domínio)
  const fromCookie = readCookie('nx_user') || readCookie('nx_lid')
  if (fromCookie) return fromCookie

  // 3. Não gerar — Yampi não é o ponto de entrada
  return ''
}
```

### Cenário Shopify+Yampi (webhook)

Quando o pedido é pago na Yampi com integração Shopify ativa, o webhook Yampi traz em `resource.metadata.data`:
- `_nx_user`, `_fbc`, `_fbp`, `_ttp`, `fbclid`, `_ga` (propagados dos `cart.attributes` do Shopify)

Isso significa que o browser script Yampi é responsável pelos eventos de **funil** (InitiateCheckout até AddPaymentInfo). O **purchase** é tratado via webhook (story 6.1).
